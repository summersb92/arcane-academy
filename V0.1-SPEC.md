# Arcane Academy — v0.1 Build Spec (handoff to project-manager)

**This is the authoritative scope for v0.1.** It deliberately carves a small, shippable, *fun* vertical
slice out of the large design corpus (`docs/00–10` + `docs/ideas/`). When this doc and the design docs
disagree, **this doc wins for v0.1**. Everything not listed under "IN SCOPE" is explicitly deferred.

- **Project root:** `C:\ClaudeSpace\arcane-academy` (greenfield — no game code exists yet; scaffold from zero)
- **Constraint:** personal project. Do **not** `git push` or create a remote without the owner's OK. Local commits are fine.

---

## 1. The v0.1 goal (one sentence)
> A player opens the page as a penniless stable-hand, discovers magic, makes a living by it, and **founds
> their academy** — a complete, self-contained **Act I** that ends on the Founding as a "to be continued."

This proves the load-bearing spine the rest of the game plugs into: the **three-zone frame**, the **theme
engine**, the **tick/offline/save engine**, and the **Task/Activity card loop**. If this slice feels good,
the game works.

**Target session:** nobody → hedge-mage → Founding in ~15–40 minutes of active play, idle-friendly throughout.

## 2. Tech stack & hard constraints
- **TypeScript + Vite + Svelte.** 100% static, no backend, no network calls, no accounts.
- Deploys as static files (GitHub Pages target; include a build that outputs to `dist/`).
- **Engine must be framework-agnostic** — `src/engine/**` must not import Svelte. UI reads state via stores
  and calls engine *actions*; UI never mutates state directly. (This keeps the sim unit-testable and the UI swappable.)
- Saves live in `localStorage` + an export/import string. No telemetry.
- Runs correctly down to **375px width** (mobile) and via keyboard.

## 3. Scope

### ✅ IN SCOPE (v0.1)
1. **The frame** — Resources panel (left), tab rail (top), Character panel (right), main content (center); responsive stack < 860px.
2. **Theme engine** — token-based; ship **3 mainstay themes: Candlelight (dark, default), Manuscript (light), High Contrast** + a **System/viewer** option that follows `prefers-color-scheme`. Picker in header; choice persisted. (Umbral optional, not required.)
3. **Tick engine** — fixed 0.1s timestep accumulator; deterministic; decoupled throttled UI render.
4. **Offline progress** — on load, simulate elapsed time (capped 12h), show a "While you were away…" summary.
5. **Save system** — **one portable versioned save format** used by every transport: localStorage autosave (~30s + on unload), clipboard **export/import string**, and **save-to-file / load-from-file**:
   - **Export to file** — downloads the save as a file (e.g. `arcane-academy-<title>-<n>.aasave`, JSON) via a Blob + object-URL download.
   - **Import from file** — a file picker (`<input type=file>` / drag-drop) reads a `.aasave`/`.json` back in.
   - Versioned with a `migrate` stub; **corruption-safe load** (validate → on failure keep the existing save and surface a clear message, never silently wipe). Serialization lives in the framework-agnostic engine (`save.ts`); the DOM download/upload is a thin UI adapter so the CLI reuses the identical format.
6. **Task / Activity system** (the Main tab) — the core loop:
   - Task types: **Instant** (one-shot), **Running** (timed Length), **Perpetual** (runs until stopped), and **Limited** (Max-capped: the Founding + fixture upgrades).
   - **Startup cost + per-second running cost**; **auto-pause** when a running cost can't be paid (resumes automatically); repeat toggle; **"At N"** repeat-scaling.
   - **Activity slots** (start with 2; one unlock raises to 3) — continuous tasks occupy a slot; instant tasks don't.
   - Rendered as **cards on a responsive grid** (Active grid + Available grid). **Whole card is the button** (click Available→start, Active→stop); locked cards dimmed & non-clickable.
7. **Resources (left panel):** Gold ⦿, Insight ◈, Renown ★, and 3 **Materials** (Moonpetal, Iron Ore, Spirit Dust). Insight has a **cap** (raised by a fixture) to exercise the caps mechanic.
8. **Character (right panel):** **Life ✚ / Stamina ⚡ / Mana ✦** vital bars (Stamina & Mana are spent/regen in v0.1; Life shown but not yet risked). **Essences:** show all 7, but only **Fire ▲** and **Dark ☾** are *awakened* (unlocked by cantrips) in v0.1; the other five appear locked/dim.
9. **Skills tab** — a small **cantrip DAG** bought with Insight (~5 nodes; see §5). First few nodes are the Act I cantrips; awakening an element cantrip starts that essence's trickle.
10. **Home tab** — a few upgradeable **fixtures** (passive bonuses); hosts the persistent **Founding progress card**.
11. **The Founding** — a Limited "Housing" task gated on Gold + Renown + Charter + Site; completing it is the **v0.1 finale**: a celebratory beat + the greyed **Academy** tab flips to "coming in v0.2." (No Act II content in v0.1.)
12. **Tabs-as-eras unfold** — rail starts as **Main only**; **Skills** appears at the spark; **Home** appears when you set up your lair; **Academy** is visible-but-greyed the whole time as the beacon. (Header holds Theme + System/save; a minimal Codex/help is optional.)
13. **Chronicle** — a simple event log on Main (task completions, unlocks, the Founding).
14. **Cheap consensus QoL** (already vetted by the councils — build these in from the start, they're load-bearing legibility):
    - **Card Payoff Preview** (hover → net "/s" projection + affordability), **Auto-Pause Explained** (paused card names the starved resource), **sourced-number tooltips** ("+2.1 = base × … "), **Caps in the left column** (amount/cap, amber at ~90%, `*` marker when a cost exceeds a cap), **Notation setting** (1.9K / 1,900 / 1.9e3), **focus-visible + keyboard tab nav**, **At-N progress chips**.
15. **Headless CLI harness** (`src/cli/`, run via `npm run cli -- <cmd>`) — an agent- and human-drivable command line over the *same engine* the UI uses (no browser, no Svelte). It is how Claude and the dev/review agents verify outcomes, balance-test, and even auto-play. Deterministic via a `--seed`. Commands:
    - `state [--json]` — dump resources, vitals, essence, tasks, phase, flags.
    - `sim <seconds> [--seed N] [--json]` — fast-forward the deterministic tick loop N seconds; print deltas + resulting state.
    - `do|start|stop <taskId>` · `learn <cantripId>` — drive task/skill actions.
    - `export` / `import <string>` — round-trip a save via the clipboard string.
    - `save <path>` / `load <path>` — write/read a `.aasave` **file** on disk in the *same* portable format the browser downloads/uploads (so a browser save loads in the CLI and vice-versa — the cross-transport check in §4).
    - `run <scenario>` — execute a scripted list of commands + **assertions** (JSON or TS); **exit code 0/1** so it doubles as a regression/balance test and is CI-able.
    - `autoplay [--goal founding] [--max-min N] [--seed N]` — a simple heuristic bot that plays toward a goal and reports whether/when it got there (validates the §4 pacing target).
    - `repl` — interactive loop for hands-on poking.
    Ships with a couple of starter scenario scripts under `scenarios/` (e.g. `founding.json`: drive Origin→Founding, assert the gate opens; `offline.json`: assert offline catch-up math).

## 4. Definition of done (acceptance criteria)
1. `npm install && npm run dev` runs; `npm run build` produces a static `dist/` that opens standalone.
2. A fresh player can go **Origin → spark → Study → lair → Scribe/Contracts → Founding** and see the finale.
3. **Reload restores exact state**; **export→wipe→import** round-trips (string); **export-to-file → wipe → import-from-file** round-trips; autosave works; a corrupt/empty file fails safe (keeps existing save + clear message). **Cross-transport:** a file the browser downloads loads via `cli load`, and a `cli save` file imports in the browser.
4. **Offline test:** close for N minutes, reopen → resources advanced correctly (capped), summary shown, no negative/NaN values.
5. A **Running** task **auto-pauses** when its cost can't be paid and **auto-resumes** — visibly explained, never reads as a bug.
6. All **3 mainstay themes** render legibly (contrast holds; Fire/Dark/Stamina/Gold visually distinct); System option follows OS.
7. Usable at **375px** and fully **keyboard-navigable** (visible focus, number-key tab jumps).
8. **Vitest** unit tests pass for: production math, offline catch-up, save round-trip + migration, a Founding-gate check.
9. No console errors; TypeScript strict; engine has zero Svelte imports.
10. **CLI harness works headlessly:** `npm run cli -- state` and `sim <sec>` run without a browser; `npm run cli -- run scenarios/founding.json` drives Origin→Founding and **exits 0** on its assertions; `autoplay --goal founding` reaches the Founding within the §4 pacing target. (The CLI importing the engine cleanly is itself the proof of the "no Svelte in engine" rule.)

## 5. Content included in v0.1 (starter data — tune freely)
Numbers are first-pass; balance to hit the ~15–40 min target. All content is **data tables** in `src/content/`.

**Cantrips (Skills DAG, cost in ◈):** `Read the Page` (opens the tree) → `Spark` (awaken ▲ Fire trickle) · `Mend` (+Stamina regen) · `Kindle Focus` (+10% all output) · `Umbral Whisper` (awaken ☾ Dark trickle).

**Tasks:**
- *Instant:* **Clean Stables** (⚡1 → ⦿2.5), **Scribe Scroll** (◈10 → Scroll, At-5:+1), **Imbue Gem (Dark)** (Gem+☾ → Dark Gem).
- *Running:* **Smith** (15s, ⚡0.4/s → ⦿5, repeatable), **Fulfil: Ward a Barn** (Contract, needs ☀? → use Fire/Dark or a student-less solo check → ⦿+★).
- *Perpetual:* **Study** (⚡0.2/s → ◈), **Rest** (→ ✚/⚡/essence regen).
- *Limited/Upgrade:* **Grand Library** (raises Insight cap → exercises caps + the `*` marker), **Widen the Study** (Activity slots 2→3).
- *Housing (finale):* **Found the Academy** (needs ⦿≥threshold, ★≥threshold, Charter, Site).

**Home fixtures:** Study Desk (◈/s), Hearth (▲ Fire/s), Ossuary (☾ Dark/s), Alembic (flavor/placeholder).
**Contracts:** 2–3 simple ones (Ward a Barn, Cleanse the Old Well) paying Gold + Renown; the Charter is a Contract/event reward or a purchasable.
**Materials:** Moonpetal, Iron Ore, Spirit Dust (drop from tasks; sink into Scribe/fixtures).

The **spark** (Origin→Awakening trigger): fires when `Gold ≥ ~25` OR a short timer elapses — guarantees progress for a purely idle player.

## 6. Architecture / suggested file layout
```
arcane-academy/
├─ index.html · vite.config.ts · tsconfig.json · package.json
├─ .github/workflows/deploy.yml        # build → GitHub Pages (owner enables Pages later)
├─ src/
│  ├─ main.ts                          # boot: load save → offline catch-up → start tick → mount UI
│  ├─ engine/                          # NO svelte imports
│  │  ├─ state.ts   (GameState: version, run{act,phase,resources,vitals,essence,tasks,skills,home,flags}, settings, lastSaved)
│  │  ├─ tick.ts    (fixed-timestep accumulator; step() runs systems in order)
│  │  ├─ systems/   (production, tasks, skills, home, founding, chronicle)
│  │  ├─ formulas.ts · offline.ts · save.ts · rng.ts · format.ts (notation) · bignum.ts (facade over number)
│  ├─ content/  (config, tasks, cantrips, elements, home, contracts, materials, themes)
│  ├─ cli/      (index.ts entry, commands/, autoplay.ts, scenario-runner.ts) — imports ONLY src/engine + src/content, never src/ui
│  └─ ui/       (App.svelte, stores.ts, panels/{Resources,Tabs,Character,Main,Skills,Home,Header}.svelte, components/{TaskCard,Meter,Tooltip,ThemePicker}.svelte)
├─ scenarios/   (founding.json, offline.json — CLI assertion scripts)
├─ tests/                              # vitest
└─ docs/ … (existing design corpus — reference, not scope)
```
`package.json` gets `"cli": "tsx src/cli/index.ts"` (tsx, or Node ≥22 native TS). The **stores bridge** (`src/ui/stores.ts`) and the **CLI** are two thin adapters over the identical engine — building the CLI early keeps the engine honest (any UI-only leak into the engine breaks the headless run immediately).

Theme tokens: `:root` = Candlelight defaults → `@media (prefers-color-scheme: light) :root:not([data-theme])` = Manuscript → `:root[data-theme="…"]` per theme. Prototyped already in `sketches/ui-mockup.html` — **use it as the visual/CSS reference**; it shows the frame, cards, themes, glyphs, and token names.

## 7. Suggested build order (for delegation)
1. **Scaffold + frame + themes** — Vite/TS/Svelte, the 3-zone shell, tab rail, theme engine + picker, static content stubs. *(Exit: themed empty frame renders, tabs switch, all 3 themes legible.)*
2. **Engine core** — GameState, tick loop, one hard-coded generator ticking a resource, save/load + export, offline catch-up. *(Exit: a number ticks; reload & offline restore correctly; tests pass.)*
3. **CLI harness core** — `src/cli/` over the engine: `state`, `sim <sec>`, `export/import`, the `run <scenario>` runner (assertions + exit code), `repl`, seeded determinism, `--json`. *(Exit: `npm run cli -- sim 60` advances state headlessly; a trivial scenario asserts & exits 0.)*
4. **Task system + Main** — task model, the 4 type behaviors, activity slots, costs/auto-pause, card grid UI, Chronicle. **+ CLI verbs** `do/start/stop` and a `tasks` scenario. *(Exit: start/stop via cards AND via CLI; auto-pause works & is explained.)*
5. **Character + Skills + resources** — vital bars, essence panel (Fire/Dark awaken via cantrips), Skills DAG, caps + `*` marker, payoff previews/tooltips. **+ CLI verb** `learn` and a skills scenario. *(Exit: cantrips awaken elements; Insight cap + storage loop legible.)*
6. **Home + the Founding + the unfold + autoplay** — fixtures, tabs-as-eras appearance, Founding gate + finale, offline summary, notation/keyboard/mobile polish. **+ CLI `autoplay --goal founding`** and `scenarios/founding.json`. *(Exit: full Origin→Founding playthrough meets §4 — verified both in-browser AND via `cli autoplay` hitting the pacing target.)*
